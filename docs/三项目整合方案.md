## 已完成阶段详情

### 阶段 0 ✅：验证地基

- `stock_full_report.py 600519` → 28 blocks，20 个有数据
- 字段名确认：`zygc`（不是 `zyge`）、`变更日期`、`主营构成` 等
- 发现问题：JSON 顶层只有 `blocks`，`basic_info` 为空，`fund_hold`/`recommend` 采集不稳定

### 阶段 1 ✅：补充数据接入

新增 2 个文件：
- `scripts/run_stock_analysis_snapshot.py` — 调 stock-analysis 采集，提取 11 个缺失 blocks
- `tradingagents/dataflows/supplemental_stock_analysis.py` — 生成中文摘要（≤2000 token）

输出：`output/supplement/{code}_stock_analysis_blocks.json`

### 阶段 1.5 ✅：数据质量标记

- supplement JSON 新增 `quality_flags`：margin → suspect，gdhs → stale，空 blocks → empty
- 摘要末尾新增"数据质量提示"，suspect/stale 数据不给强结论

### 阶段 2 ✅：接入 Agent

修改 4 个文件：
- `agent_utils.py` — 新增 `@tool get_supplemental_data`
- `trading_graph.py` — fundamentals/lockup ToolNode 接入
- `fundamentals_analyst.py` — import + tools + prompt + 质量约束
- `lockup_watcher.py` — import + tools + prompt + 质量约束

关键发现：ToolNode 是显式列表，不是自动发现。

### 阶段 2.5 ✅：真实调用验证

验证方式：`TradingAgentsGraph.propagate()` 直接调用 DeepSeek API。

结果：
- fundamentals_analyst 实际调用了 get_supplemental_data（主营构成、股本结构、十大股东、分红、业绩预告）
- lockup_watcher 实际调用了 get_supplemental_data（十大股东变动、股本结构）
- 报告中原文："数据质量说明：融资融券数据标记为可疑，股东户数数据标记为过旧，均已降权处理，不纳入分析结论"

附带修复：PDF 导出 Windows 中文字体支持。

### 阶段 3 ✅：统一 HTML 报告

新增 `scripts/render_unified_html.py`（约 240 行）。

设计要点：
- MD 使用 `## Step N:` 格式对齐 html_renderer 的 `parse_step_sections()` 解析器
- JSON adapter 补齐 `code`/`name`/`blocks`，basic_info 为空时自动注入
- `sys.path` 用绝对路径挂载 stock-analysis/src

输出：
```
output/unified/{code}_unified.md
output/unified/{code}_unified.json
output/reports/{code}_unified.html
```
