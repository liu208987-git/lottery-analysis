## 二、根因分析

### 原代码缺陷

**`already_sent()` 原始实现（第715-729行）：**
```python
def already_sent(kind: str, h: str) -> bool:
    log_path = PUSH_DIR / "send_log.jsonl"
    if not log_path.exists():
        return False
    today = today_str()
    try:
        for line in log_path.read_text(encoding="utf-8").splitlines():
            if not line.strip():
                continue
            item = json.loads(line)
            if item.get("date") == today and item.get("kind") == kind and item.get("hash") == h and item.get("ok"):
                return True
    except Exception:
        pass
    return False
```
- 无任何互斥机制，并发安全为零
- 读取和写入之间没有原子性保证

**`append_log()` 原始实现（第732-746行）：**
```python
def append_log(kind: str, h: str, ok: bool, detail: str = ""):
    ...
    try:
        with log_path.open("a", encoding="utf-8") as f:
            f.write(json.dumps(item, ensure_ascii=False) + "\n")
```

- 写入后没有 `flush()` 和 `fsync()`，数据滞留在内核 Page Cache 中
- 并发进程通过 `read_text()` 读取时可能看不到最新行

## 三、修改内容

### 修改1：新增文件锁机制（第806-853行）

在 `already_sent()` 之后、`append_log()` 之前，新增三个函数：

```python
_LOCK_DIR = BASE / "output" / ".push_locks"
_LOCK_DIR.mkdir(parents=True, exist_ok=True)


def _push_lock(kind: str, h: str) -> str:
    """简易文件锁，防止并发重复推送。返回锁文件路径。"""
    os.makedirs(str(_LOCK_DIR), exist_ok=True)
    lock_path = _LOCK_DIR / f"{kind}_{h}.lock"
    return str(lock_path)


def acquire_push_lock(kind: str, h: str, timeout: float = 3.0) -> bool:
    """获取推送锁，成功返回 True，超时返回 False。
    
    使用 os.open(O_CREAT | O_EXCL) 原子操作，Linux 内核级互斥。
    """
    lock_file = _push_lock(kind, h)
    deadline = time.time() + timeout
    while time.time() < deadline:
        try:
            fd = os.open(lock_file, os.O_CREAT | os.O_EXCL | os.O_WRONLY)
            with os.fdopen(fd, "w") as f:
                f.write(f"{kind}/{h} locked by pid {os.getpid()}\n")
            return True
        except FileExistsError:
            # 检查锁是否过期（超时后强制获取）
            if time.time() > deadline - 2:
                try:
                    mtime = os.path.getmtime(lock_file)
                    if time.time() - mtime > timeout:
                        os.unlink(lock_file)
                        continue
                except OSError:
                    pass
            time.sleep(0.2)
    return False


def release_push_lock(kind: str, h: str):
    """释放推送锁。"""
    lock_file = _push_lock(kind, h)
    try:
        os.unlink(lock_file)
    except OSError:
        pass
```

**设计要点：**
- 锁文件存放在 `output/.push_locks/`，不影响 send_log.jsonl 正常读取
- `os.open(O_CREAT | O_EXCL)` 是 POSIX 原子操作，没有竞态窗口
- 超时机制防止僵尸锁导致永久阻塞
- 过期锁强制清理（等待超时时间的 2/3 后检查 mtime）

### 修改2：`append_log()` 加 fsync 强一致性（第861-864行）

```python
try:
    with log_path.open("a", encoding="utf-8") as f:
        f.write(json.dumps(item, ensure_ascii=False) + "\n")
        f.flush()          # ← 新增：刷新 Python 缓冲区
        os.fsync(f.fileno())  # ← 新增：强制写入磁盘
except Exception as e:
    print(f"[WARN] 写入发送日志失败: {e}")
```

**设计要点：**
- `flush()` 确保 Python 的 io 缓冲区写入系统内核缓冲区
- `os.fsync(f.fileno())` 确保内核缓冲区写入物理磁盘
- 并发进程通过 `read_text()` 读取时保证看到最新数据

### 修改3：`--stdout` 模式加锁保护 + 详细日志（第1098-1119行）

修改前：
```python
if args.stdout:
    report_path = PUSH_DIR / f"{kind}_report.md"
    write_file(report_path, text)
    h = msg_hash(text)
    if not args.force and already_sent(kind, h):
        print(f"[跳过] 今日已推送过相同内容", file=sys.stderr)
        sys.exit(0)

```

修改后：
```python
if args.stdout:
    report_path = PUSH_DIR / f"{kind}_report.md"
    write_file(report_path, text)
    h = msg_hash(text)
    # 加锁防止并发重复推送
    if not args.force and already_sent(kind, h):
        print(f"[跳过] 今日已推送过相同内容", file=sys.stderr)
        sys.exit(0)
    if not acquire_push_lock(kind, h, timeout=5.0):
        print(f"[跳过] 推送锁获取失败（可能正在推送中）", file=sys.stderr)
        sys.exit(0)
    try:
        # 再次检查（持有锁后二次确认）
        if not args.force and already_sent(kind, h):
            print(f"[跳过] 二次检查已推送过", file=sys.stderr)
            sys.exit(0)
        # 日志包含内容长度和预览摘要
        preview = text.replace("\n", "\\n")[:60]
        append_log(kind, h, True, f"hermes deliver=origin | len={len(text)} | preview={preview}")
        print(text)
    finally:
        release_push_lock(kind, h)
    sys.exit(0)
```

**设计要点：**
- 持有锁后**二次检查** `already_sent()`，消除并发写入窗口
- `finally` 确保锁一定会释放，不因异常导致死锁
- detail 改为包含 `len=`（内容长度）和 `preview=`（内容前60字符预览）
